# ImageExcute
测试opencv用
包含了读取图像，抖动图像以及字符画的示例
关于字符画
这个测试中所使用的是8点盲文，遇到了不少问题

1.二值化的算法选择
-
	由于要做字符画，所以需要二值化图像，然而二值化算法的泛用行都不强，需要自己处理这些情况
	例如：otsu算法适合二次元图像，相对于复杂度高的现实图像，效果就不太好

2.8点盲文utf8的字符宽度不统一
-

	从utf-2801 - utf-28ff的字符宽度都是一致的，但唯独utf-2800（相当于空格）的字符宽度与其他的不统一
	这就导致了二值化不均的一行会对的十分不齐,输出的结果相当不规整
3.字符画的输出
---
	终端不支持直接输出8点盲文字符，所以只能输出到txt文件中
# 运行
简单写了一个processbar的简易进度条，可以简单运行并计算进度
```
ProcessData pd = new ProcessData(0);
ProcessBar pb = new (() => {retrun Task(pd);} , pd);
pb.Start();
```
